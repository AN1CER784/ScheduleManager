# AGENTS.md

Чек-лист качества для выполнения задач

## Общие критерии результата

- [ ] **Оптимальные запросы** (нет лишних обращений к БД/внешним сервисам, нет N+1, адекватные индексы/выборки).
- [ ] **Результат задачи достигнут** — постановка могла сместиться, поэтому свериться с **последними комментариями** в задаче.

## Обработка ошибок и устойчивость

- [ ] В коде есть `try/except` там, где это оправдано; обработаны типовые исключения (`KeyError`, `DoesNotExist`, и т. п.).
- [ ] Присутствует **широкий `except`** с **логированием ошибки** (чтобы не терять неожиданные падения), но без «глушения» критичных проблем там, где нужно падать.
- [ ] Для `dict.get(...)`: либо передано значение по умолчанию, либо дальше обработан кейс, что переменная может быть `None`.
- [ ] Для `GET`-эндпоинтов: обработано, что объект **не существует** и что вернулось **больше одного** (`MultipleObjectsReturned`).
- [ ] Для `filter(...)`: есть проверка/логика на случай **пустого результата**.

## Безопасность и авторизация

- [ ] Любой запрос/метод, который возвращает данные клиенту, учитывает **авторизацию пользователя**.
- [ ] Нет возможности получить **чужие данные** (проверки портала/владельца/прав доступа везде, где нужно).

## Производительность и работа с БД

- [ ] Из БД достаются только **необходимые поля** (например, `only()`, `values()`, `select_related/prefetch_related` по ситуации).
- [ ] По возможности в циклах **нет обращений к БД** (заранее вытянуто всё нужное, нет N+1).

## Качество кода и поддерживаемость

- [ ] Нет логических ошибок, все импорты нужны, переменные используются.
- [ ] В коде есть комментарии на РУССКОМ там, где без них сложно понять «почему так».
- [ ] Каждый метод задокументирован на РУССКОМ (docstring или понятное описание назначения/контракта).
- [ ] Методы переиспользуются, а не копируются: всё, что можно вынести в общий (пусть маленький) метод/функцию — вынесено.
- [ ] PyCharm ничего не подчёркивает (максимум — замечания по комментариям/стилю).
